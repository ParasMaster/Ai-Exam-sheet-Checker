AI-Powered Answer-Sheet Grading System

Previous Model Overview & Current Project Plan

1. Previous Model Overview

The previous prototype implemented:

  • Basic OCR Integration using open-source Tesseract for printed text only.
  • Keyword Matching for short-answer grading, relying on exact word overlap.
  • Manual Workflow where low-match answers were flagged for teacher review.
  • Limitations:
    • Poor handwriting accuracy (≈70%).
    • Inflexible to synonyms or paraphrasing.
    • No confidence scoring or semantic analysis.

2. Current Project Plan

2.1. Goals

  • High-accuracy OCR for both printed and handwritten text.
  • Semantic answer understanding to allow paraphrasing.
  • Automated grading with confidence scoring.
  • Human-in-the-Loop review for low-confidence cases.

2.2. Architecture

  1. Client UI
    • Web form for uploading PDFs/images of answer sheets
    • Status dashboard for processed and flagged items
  2. Backend Services
    • OCR microservice (e.g., Google Vision + custom HTR model)
    • Semantic analysis service using transformer models
    • Grading engine with configurable rubrics & confidence thresholds
    • Manual‐review queue and annotation interface
  3. Data Storage
    • Document store (MongoDB) for scanned images and text
    • Relational DB (PostgreSQL) for grades and audit logs
  4. DevOps & Scaling
    • Containerized services (Docker/Kubernetes)
    • Asynchronous processing via message queues (RabbitMQ)
    • Auto‐scaling on cloud infrastructure

2.3. Milestones & Timeline

Phase Tasks Duration
Phase 1 OCR & Text Extraction Prototype 2 weeks
Phase 2 Semantic Analysis & Similarity Matching 3 weeks
Phase 3 Automated Grading Engine & Confidence Scoring 3 weeks
Phase 4 Human Review Interface & Workflow 2 weeks
Phase 5 Performance Tuning & Deployment 2 weeks

2.4. Technology Stack

  • Frontend: React.js or Angular
  • Backend: Node.js + Express.js (or Django)
  • OCR & NLP: Google Cloud Vision, HuggingFace transformers
  • Databases: MongoDB, PostgreSQL, Redis
  • DevOps: Docker, Kubernetes, RabbitMQ
